# 晋升评估不向 360°评审员开放（已被 ADR-0066 取代）

> 状态：本 ADR 已被 ADR-0066 取代。晋升改为员工本人提交申请表单后发起的独立流程，不再使用绩效评估角色或提交模型；未来晋升流程参与角色另行定义。

现有产品文档和代码曾允许 360°评审员填写晋升反馈。我们重新划定职责：晋升评估子表单只包含员工和 Leader 两个角色区段，员工区段嵌入员工自评提交，Leader 区段嵌入上级评估提交；360°评审员不参与被评估人的晋升评价，360°表单、提交和结果中都不出现晋升评估项。晋升评估仍随绩效周期启动，并只对启用晋升评估的参与者显示，不产生独立提交流程。

**Considered Options**：继续收集 360°评审员的晋升意见可以增加观察视角，但会把常规协作反馈与正式晋升判断混在同一职责中，并扩大晋升敏感信息的填写范围。将晋升评价限定为员工陈述和 Leader 判断，使 360°评审员只承担常规绩效反馈职责。

**Consequences**：需要删除现有 `promotionFeedback` 及评审员晋升表单入口，模板设计器的晋升子表单只展示员工与 Leader 两个角色区段；产品详细设计文档中“评审员填写晋升反馈”的旧描述后续必须同步修订。
