# 周期归档必须人工执行且全员收口

绩效周期不自动归档。只有权限范围覆盖该周期全部参与者的 HR，或全局 Admin，可以手动归档。归档前每名参与者必须处于 `CONFIRMED`、`NO_RESULT` 或 `WITHDRAWN` 之一；所有有结果参与者必须存在当前有效的校准结果和结果版本。不得存在待确认、申诉中、再次确认中、周期退回后尚未重新校准、缺少上级评估或其他未收口参与者。归档页面展示参与人数、等级分布、无结果人数、中途退出人数与全部阻塞项，并要求二次确认。成功后冻结周期全部数据，记录操作人、时间和归档摘要。

**Considered Options**：满足条件后自动归档可以减少操作，但会让 HR失去最终核对等级分布和异常人员的控制点；只允许 Admin又会让日常周期运营过度集中。允许覆盖全员范围的 HR与 Admin人工归档，在权限完整的前提下保留业务确认。

**Consequences**：归档接口需要计算全量收口检查并验证 HR对周期所有参与者的对象级权限，不能只检查静态角色。归档摘要应作为不可变快照保存；任何阻塞项都返回具体参与者和原因，前端禁止绕过。
