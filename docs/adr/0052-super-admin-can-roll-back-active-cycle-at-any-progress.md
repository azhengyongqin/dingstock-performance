# 超级管理员可以在任意进度退回进行中周期

`ACTIVE` 周期可以退回 `SCHEDULED` 或 `DRAFT`，且不受已有草稿、正式提交、校准、结果推送、员工确认或申诉进度限制。该高风险操作只能由超级管理员执行，必须填写原因并二次确认，所有既有业务数据与审计记录都不得物理删除。已经向员工推送的当前结果版本必须标记为失效，并向相关员工发送“绩效周期已退回、原结果已失效”的飞书通知。

**Considered Options**：有校准或推送后禁止周期退回可以保持结果链稳定，但无法满足严重配置或流程错误下整体回退重做的管理需要。允许 HR 或 Leader操作又会把影响全周期与员工结果的权限扩得过大。限定超级管理员并要求结果失效通知，可以保留纠错能力和明确责任。

**Consequences**：周期状态机需要支持 `ACTIVE → SCHEDULED` 与 `ACTIVE → DRAFT`。退回操作必须以事务或可恢复流程暂停任务、失效当前结果版本、处理确认与申诉关联并写审计；通知需要记录发送状态并支持失败重试。评估锁定、校准记录和参与者主状态按 ADR-0053 恢复。
