# 绩效周期不配置考核期间起止日期

绩效周期名称直接表达业务期间，例如“2026 上半年绩效评定”，系统不再要求配置员工工作成果对应的考核开始日和结束日。需要精确管理的是评审流程本身：周期保存计划启动时间，并根据配置模板的相对日程预设生成自评、360°、上级评估、校准、确认等阶段窗口。

**Considered Options**：同时保存考核期间与评审流程日期，数据更结构化，但当前业务只需要通过周期名称识别上半年或下半年，额外日期会增加创建步骤并与流程窗口产生语义混淆。保留名称与流程时间即可满足现行操作。

**Consequences**：当前 `PerfCycle.startDate`、`PerfCycle.endDate` 及创建页面的考核期间起止日期需要移除或迁移，不能再用它们表达业务期间。列表、报表和历史展示以周期名称标识业务期间；任务开放读取实际开始时间，填写提醒读取软截止时间，二者语义按 ADR-0041 与 ADR-0051 区分。
