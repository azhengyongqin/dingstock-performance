# 加权子表单显式标记核心维度

绩效等级计算方案中的“维度一”不是数组或页面排序意义上的第一项，而是承担强制 C 与最高等级封顶约束的核心业务维度。每个采用加权评级或加权评分的评估子表单必须且只能显式标记一个核心维度；普通岗和管理岗默认都将“核心业绩”标记为核心维度。调整显示顺序不改变核心身份，核心标记随评估表单模板复制为周期快照。

**Considered Options**：按 `sortOrder = 0` 或数组第一项识别核心维度无需新增语义字段，但拖拽排序、插入说明维度或模板复制后都可能悄然改变约束对象。通过名称匹配“核心业绩”同样会受改名和多语言影响。显式标记才能让计算、界面和审计使用稳定标识。

**Consequences**：评估维度配置需要核心维度属性；模板可用性与周期启动检查必须验证每个加权子表单恰有一个核心维度。删除或取消核心维度前必须重新指定，否则配置不可用于周期；计算明细需要记录触发约束的核心维度快照。
