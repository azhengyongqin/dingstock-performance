# 360°评审员关系加权并随模板复制为周期快照

360°评估不对所有评审员直接等权，而是先在同一评审员关系内对多名评审员的结果等权平均，再按关系基础权重汇总。默认基础权重为组织负责人 30%、项目负责人 30%、同部门同事 25%、跨部门协作方 15%；没有评审员的关系不参与计算，其余有效关系按各自基础权重占有效基础权重总和的比例归一化至 100%。直属上级不进入 360°计算，因为其评价由上级评估阶段独立承载。

这组基础权重属于配置模板，而不是全局实时配置。管理员在模板管理中维护权重；创建周期或重新套用模板时，权重与评估规则、评估维度一并复制成周期配置快照。管理员可以修改当前周期的权重，但修改必须填写原因并留下审计，且不回写配置模板。

**Considered Options**：所有评审员直接等权无法体现不同评价视角的重要性；为单个评审员设置权重会增加人为操纵空间和维护成本；使用全局实时权重会导致一次系统设置修改改变进行中甚至历史周期的计算口径。模板与周期快照兼顾复用、差异化与结果可追溯性。

**Consequences**：模板管理与周期配置需要新增 360°评审员关系权重编辑界面；计算记录必须从关系化的评估项结果汇总，并保存实际参与的关系、关系内平均结果、归一化后权重与最终汇总结果。模板可用性和周期启动检查还需要覆盖关系权重配置的合法性。
