# 配置模板编排可复用的评估表单模板（已被 ADR-0066 部分取代）

> 状态：关于配置模板编排绩效评估表单模板的决定继续有效；关于评估表单模板包含晋升子表单、晋升内容嵌入员工自评和上级评估的决定已被 ADR-0066 取代。晋升申请表单改由独立晋升流程使用。

配置模板与评估表单模板是两个领域对象。配置模板保存周期级评估规则、360°评审员关系权重，并选择一组可复用的评估表单模板；评估表单模板按适用人群打包员工自评、360°评估、上级评估和晋升评估四个子表单，每个子表单分别保存评估维度和评估项，默认提供普通岗（D）与管理岗（M）两套。晋升评估子表单只对周期内启用晋升评估的参与者生效，不新增独立周期阶段；它只包含员工和 Leader 两个角色区段，分别嵌入自评与上级评估，360°评审员不参与晋升评价。HR 创建绩效周期时只选择一份配置模板，不需要再次逐个选择表单模板；系统把配置模板及其所选表单模板完整复制为周期快照。

**Considered Options**：继续把周期规则、D/M 维度和所有评估项塞进一个模板，编辑入口较少，但表单无法独立复用，岗位差异只能靠同一份大模板中的条件字段维持；把每个评估阶段拆成独立模板会让配置模板需要组合 D/M 与多个阶段的大量模板；创建周期时分别选择配置模板和多个表单模板则把编排复杂度重复转嫁给每次建周期的 HR。由一份人群表单模板打包四个子表单，再由配置模板预先组合人群模板，可以在复用能力和创建体验之间取得平衡。

**Consequences**：数据库需要配置模板版本与评估表单模板版本的关联关系；配置模板发布校验必须确保所选表单模板版本已发布且完整、职级前缀不重叠、适用范围合法。模板管理需要分别提供周期规则编辑和表单设计能力，同时在配置模板中提供表单模板版本选择及职级覆盖范围预览。
