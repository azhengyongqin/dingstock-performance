# 校准前更换 Leader 时转移上级评估编辑权

参与者尚未形成首次校准结果时，HR 更换考核 Leader后，旧 Leader立即失去上级评估修改权，新 Leader获得当前上级评估答卷的编辑权。旧 Leader已经成功提交的内容继续作为当前有效评价并保留原提交人标识，避免更换责任人后阶段结果突然缺失；新 Leader可以沿用，也可以修改并重新提交。新 Leader重新提交后，当前答卷内容和归属更新为新 Leader并触发阶段重算，不保留旧答卷内容版本，但保留 Leader更换和重新提交的审计事件。参与者已经校准后，更换 Leader不能解除评估锁定或改变上级评估内容。

**Considered Options**：更换 Leader时直接清空旧评估会让已完成输入和阶段结果消失；让旧 Leader继续拥有修改权则与新的责任快照冲突；要求新 Leader必须无修改重新提交又会制造无意义操作。保留当前有效内容并转移编辑权，能同时保持结果连续性与责任归属。

**Consequences**：上级评估需要区分当前责任 Leader与最后成功提交人。更换接口要原子更新 Leader快照、任务归属和对象级权限；新 Leader首次打开时应提示当前内容由原 Leader提交。重新提交后更新提交人、提交时间和阶段结果，不建立答卷历史版本。
