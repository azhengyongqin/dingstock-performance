# 每个常规评估维度仅允许一个计分项

加权评级或加权评分子表单中，每个常规评估维度必须且只能配置一个与阶段结果模式匹配的计分项：加权评级使用一个评级选择项，加权评分使用一个 0～100 分评分项。同一维度可以附加多个评语、Markdown、附件、链接等非计分项。阶段计算只配置维度权重，不再增加评估项权重。如果业务需要多个独立分项参与计算，应将它们拆成多个常规评估维度。

**Considered Options**：允许一个维度内存在多个计分项需要再引入评估项权重、缺失项归一化和双层约束解释，配置与结果展示都会明显复杂化。只允许一个计分项与目标规则中“核心业绩 70% 对应一个业绩分数”等结构一致，也让维度成为稳定且可解释的计算单元。

**Consequences**：表单模板发布校验必须保证每个常规维度恰好有一个计分项、输入类型与子表单模式一致，并禁止计分项配置权重；非计分维度不得包含计分项。模板编辑器应引导管理员通过新增维度表达新的计分因子。阶段计算可以直接读取每个维度唯一计分项的计算分。
