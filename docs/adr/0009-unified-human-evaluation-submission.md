# 三类人工评估统一为评估提交模型

员工自评、360°评估与上级评估都由填写人提交一组模板定义的评估项结果。我们决定用统一的评估提交父记录承载三类人工答卷，以评估阶段区分类型，并用关系化评估项结果作为子记录。父记录统一保存参与者、阶段、填写人、状态与提交时间；360°提交额外关联有效评审员指派及评审员关系，上级评估的填写人必须匹配考核 Leader 快照，自评与上级评估分别保证每名参与者至多一份。AI 评估不是人工填写，继续由独立异步 AI 报告承载，但必须输出统一的阶段结果。

**Considered Options**：保留 `PerfSelfReview`、`PerfReview`、`PerfManagerReview` 三套独立提交模型，能减少当前代码迁移量，但模板化评估项会迫使三套表单分别实现渲染、校验、保存、提交和计算，且固定自评字段会持续限制管理员配置表单。统一提交模型把阶段差异收敛为显式约束，同时保留 AI 异步任务与人工答卷之间真实存在的生命周期差异。

**Consequences**：数据库需要评估阶段枚举、统一评估提交表和评估项结果表，并为自评、360°与上级评估设计明确的业务唯一性和索引；现有三类提交数据及接口需要迁移或提供兼容层。自评总结、综合评语以及员工/Leader 的晋升材料应逐步转为模板定义的评估项，而不是继续扩充阶段专用固定列；360°提交不包含晋升评估项。
