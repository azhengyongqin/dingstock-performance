# AI 报告完全异步生成，不是流程阶段

产品文档曾将「AI 分析中」设计为周期状态机的独立阶段（`AI_ANALYZING`/`AI_DONE`），意味着全员报告生成完毕才能进入校准。我们决定：**AI 报告不是流程阶段**——从周期与参与者状态机中删除 AI 状态；报告按人异步触发（上级评估提交即入队，任务表 + cron + Redis 锁），生成完自动出现在校准页，失败显示标记、可手动重新生成，任何情况下不阻塞任何环节。

**理由**：AI 报告与 360°一样是参考信息（见 ADR-0001，等级是人工判断），参考信息不应拥有让流程停摆的权力；把绩效日程与外部 LLM 服务的可用性解耦。LLM 接入经 Vercel AI SDK（ai-sdk.dev）抽象，供应商/模型是配置问题而非代码问题——绩效数据属敏感员工个人信息，供应商选择需过数据合规判断。

**Consequences**：`REVIEWING → CALIBRATING` 直达；校准页需容忍报告「生成中/失败/缺失」三态；「Leader 必须查看 AI 报告后确认」降级为前端引导。可见性与 360°对齐：管理链实名可见、员工不可见（AI 报告是 360 明细的浓缩转述，员工可见会绕穿 360 保密规则）。
