# 360°结果先按关系聚合再应用维度权重

360°正式结果采用固定聚合顺序。首先将每位已提交评审员在每个常规维度填写的原始评级或评分转换为统一计算分；然后在同一评审员关系内，按维度对多名评审员的计算分做算术平均。排除没有正式提交的关系后，将剩余有效关系的基础权重归一化，并按关系权重计算每个维度的关系加权分。最后按维度权重计算 360°阶段综合得分，按统一精度规则四舍五入至两位小数、映射初始等级，再应用核心维度等约束形成阶段等级。加权评级模式下，聚合后的维度计算分按评级区间映射为维度等级供约束规则使用，单个评审员给出的低评级不会直接替代聚合后的维度等级。

**Considered Options**：先计算每名评审员的个人综合分再按关系平均，在所有表单完整时可能得到相同综合分，但无法稳定保留“某一维度在各关系中的贡献”，也不利于核心维度约束与解释。以单个评审员的最低评级直接触发整体约束会让一名评审员拥有事实上的否决权，与关系平均规则冲突。先按关系聚合每个维度，再应用维度权重，能保留完整的可解释计算链。

**Consequences**：360°计算明细需要保存评审员计算分、关系内维度均分、有效关系及归一化权重、最终维度分、阶段综合分、初始等级、约束原因与阶段等级。计算服务必须按该顺序执行并使用精确小数，不得在中间步骤舍入。预览和正式结果复用同一算法，仅数据范围与结果状态不同。
