# 可计算的评估项结果使用关系化明细

评估模板由评估维度和评估项组成，评估人实际填写的是评估项。我们决定不再把所有可计算结果只保存在评估记录的 `dimensionScores` JSONB 数组中，而是将每个评估项结果保存为可关联、可查询的独立明细记录。评级型结果同时保存原始评级和按当期规则得到的计算分；评分型结果同时保存原始分数和计算分；文本、Markdown、附件等非计分结果可以保留结构化载荷。

**Considered Options**：继续使用单个 JSONB 数组改表较少，也适合完全不可预知的表单载荷，但 360°关系加权、同关系平均、维度汇总、规则约束、报表筛选和结果复算都会频繁读取其中的评分字段。把需要计算和查询的数据隐藏在 JSONB 中，会失去关系约束、索引能力与清晰的数据血缘。关系化明细只固定通用结果语义，动态表单配置与非计分载荷仍可使用 JSONB。

**Consequences**：后续数据库设计需要按 ADR-0009 建立统一评估提交及评估项结果子记录，并明确评估项、评估维度、原始值、计算分、计分方式快照与规则快照的关联。现有 `PerfReview.dimensionScores`、`PerfManagerReview.dimensionScores` 以及依赖它们的前后端代码需要迁移；最终修改 Prisma 时必须同步更新 `docs/数据库表结构.md`，并为常用汇总路径设置唯一约束和索引。
