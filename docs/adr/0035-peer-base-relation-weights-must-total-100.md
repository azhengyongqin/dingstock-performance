# 360°基础关系权重必须严格合计 100%

参与 360°计算的组织负责人、项目负责人、同部门同事、跨部门协作方四种关系类型是固定受控词表，不允许管理员新增、删除或改名。每项基础权重必须大于 0%、不超过 100%，最多两位小数，四项合计必须严格等于 100%；不满足时配置模板不能发布，周期配置也不能保存。有效关系归一化仅用于某些关系没有正式提交的运行时场景，不能用于容忍或修复基础权重错误。如果某位参与者不需要某类关系，应不指派该类评审员，而不是把全局基础权重配置为 0%。

**Considered Options**：只把四项数值视为任意比例并在每次计算时统一归一化，可以减少配置校验，但页面显示为百分比时会让管理员误以为其合计代表完整分配；允许 0% 又会产生“评审员完成了任务但结果完全不计入”的隐性行为。要求正权重且合计 100%，能让配置含义直接、可审计，运行时归一化只承担缺少有效关系这一项职责。

**Consequences**：配置模板发布、周期创建和周期配置修改必须使用精确小数验证四项权重范围、精度与总和。编辑页面需要实时展示合计值和差额，并在不等于 100% 时阻止保存或发布。关系类型应使用固定枚举或受控字典，不能由租户数据任意扩展。
