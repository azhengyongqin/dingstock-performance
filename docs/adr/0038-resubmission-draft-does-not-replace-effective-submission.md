# 二次编辑草稿不替换当前有效提交

评估人已有一次成功正式提交后再次进入编辑，上一份成功提交的内容继续作为当前有效评价参与计算。后续修改先保存为临时草稿，不立即改变正式评估项结果或阶段结果；只有评估人点击重新提交且完整性校验通过后，系统才原子替换当前有效答卷并触发相关重算。填写提醒截止时间不会清除草稿或阻止重新提交；参与者完成校准并评估锁定时，仍未重新提交的更新草稿保持草稿状态、不生效，继续以上一次成功提交为准。

**Considered Options**：进入编辑或自动保存草稿时立即覆盖正式内容，会让不完整填写参与计算，并可能仅因打开编辑页面就使既有结果失效。再次编辑时把正式提交改回未提交状态，又会让流程进度反复倒退。让上一份提交持续生效、重新提交时原子替换，能保持流程和结果稳定。

**Consequences**：系统需要区分一份当前有效提交与一份可清理的临时编辑草稿，但不建立历史版本链。自动保存只写临时草稿；重新提交事务需要完成校验、替换当前评估项结果、更新提交时间、清除草稿并触发重算。进度统计始终以是否存在成功提交为准，不因二次编辑草稿而倒退。
