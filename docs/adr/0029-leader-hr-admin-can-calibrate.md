# 直属 Leader、HR 与 Admin 均可形成校准结果

校准最终确认权默认开放给三类主体：直属 Leader 可以校准其考核 Leader 快照范围内的参与者；HR 可以在其 RBAC 授权组织范围内校准；Admin 可以全局校准。三类主体都可以提交“维持”或“调整”的正式校准结果，不把 Leader 限制为只能提出建议。校准继续采用 append-only 记录，后续重新校准追加新决定而不覆盖历史。

**Considered Options**：只允许 HR/Admin 最终确认能强化集中控制，但会让团队校准全部依赖 HR 操作，与直属 Leader 承担绩效管理责任不符；允许任意管理者校准又会扩大敏感权限。以考核 Leader 快照和 HR 组织授权作为范围边界，既允许业务负责人直接完成校准，也保持最小权限。

**Consequences**：校准接口不能只用静态 `HR/ADMIN` 角色守卫，必须结合参与者的考核 Leader 快照与 HR 组织范围做对象级授权；校准工作台按权限返回可见参与者。审计记录必须保存操作人身份、权限来源与作用范围。
