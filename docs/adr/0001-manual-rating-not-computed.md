# 绩效评级由 Leader 人工判定，不做算法聚合

绩效维度带有 `weight` 字段，产品文档 §9.2 曾写「维度权重用于计算最终建议等级」，很容易让人以为评级应由 360°评分与维度权重加权算出。我们决定：**评级是管理判断，不是算术结果**——直属 Leader 在上级评估中人工选定初始评级，校准环节负责跨团队拉齐；360°评分、自评、维度权重一律只作参考信息展示，不参与任何计算。`weight` 字段保留但仅用于前端展示排序/重要性标注。

**Considered Options**：系统按权重加权算出「建议评级」供 Leader 覆盖（需定义 360 多人合并、角色权重、分值→评级映射等一整套算法，且诱导评分博弈）；纯算法定级（与校准会议的人工判断根本冲突）。选人工定级符合多数公司绩效实践，也让「启动后配置全部可改」变得安全——中途改权重不影响任何已有结果。

**Consequences**：校准页只展示当前评级分布，不提供预设占比或自动拦截；看到 `weight` 未参与计算的读者请勿"修复"——这是刻意的。
