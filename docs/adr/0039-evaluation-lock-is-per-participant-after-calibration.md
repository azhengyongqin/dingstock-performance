# 评估答卷在单个参与者完成校准后锁定

评估修改边界按参与者独立判断。某参与者首次形成正式校准结果之前，其员工本人、各 360°评审员和考核 Leader 仍可分别修改并重新提交自己的当前答卷；该参与者一旦形成正式校准结果，其员工自评、全部 360°评估和上级评估立即锁定，不能再修改或重新提交。不需要等待周期内其他参与者完成校准。已锁定参与者如需改变最终结果，通常只能追加重新校准，不能回改评估答卷；按 ADR-0053 执行超级管理员周期退回时，校准失效并整体解除锁定。普通周期配置重算不解除该锁定，也不自动修改既有校准结果。

**Considered Options**：在周期进入校准阶段时统一锁定所有答卷，会阻止尚未校准人员继续补充评价；等整个周期校准完成后统一锁定，又会让已经确认校准的员工继续受到输入变化影响。按参与者在形成校准结果时锁定，可以让校准决定建立在稳定输入上，同时不妨碍其他员工继续完成评估。

**Consequences**：参与者需要可原子判断的评估锁定状态或等价的首条有效校准存在性约束。创建首次校准结果与锁定操作必须在同一事务内完成；所有自评、360°和上级评估保存、提交接口都必须进行对象级锁定检查。校准工作台可以混合展示“待校准且可更新”与“已校准且已锁定”的参与者。
