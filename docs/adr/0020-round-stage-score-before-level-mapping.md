# 阶段综合分舍入两位后映射等级

评审人填写的评分范围为 0～100，最多两位小数；原始分数、评级映射分和全部中间计算使用精确十进制，不使用二进制浮点数，也不在关系平均、关系归一化或维度加权过程中提前舍入。最终阶段综合分按四舍五入保留两位小数，再依据评级区间映射初始等级；未舍入计算值和过程明细仍保留用于审计。

**Considered Options**：按未舍入值映射等级更贴近原始数学结果，但可能出现内部值 79.995 映射为 B、页面却显示 80.00 的口径冲突；每一步都舍入则会积累误差并让计算顺序影响结果。只在最终阶段综合分舍入一次，再映射等级，使用户看到的分数和等级边界一致。

**Consequences**：数据库评分字段应使用 PostgreSQL `numeric`/Prisma `Decimal`，后端计算必须使用十进制运算库并明确四舍五入模式，不能直接依赖 JavaScript `number`。计算结果需要同时保存未舍入明细、两位阶段综合分及由其映射的等级，前后端测试必须覆盖 59.995、79.995、89.995 等边界。
