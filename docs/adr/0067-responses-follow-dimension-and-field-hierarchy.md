# 作答按维度与字段两层关系化保存

评估提交下的作答采用“维度作答 + 字段作答”两层关系模型。同一提交对同一维度至多一条维度作答；计分维度的原始评级或原始分数及计算分直接保存在维度作答。每个已填写的非计分表单字段保存一条从属于维度作答的字段作答；没有字段且只展示说明的非计分维度不产生作答。评级和分数不再伪装成表单字段，也不再使用统一评估项结果承载两种语义。

**Considered Options**：继续复用统一评估项结果表改动较小，但需要为维度计分构造不存在的评级或分数字段，领域模型与存储继续错位；把字段内容整体放进维度 JSONB 虽结构简单，却削弱字段级唯一约束、类型校验和迁移匹配。两层关系表与表单定义层级一致，并保留可查询计算结果和柔性字段载荷。

**Consequences**：提交、草稿、完整性校验、预填迁移、阶段计算和报表查询都必须改用维度作答与字段作答。数据库应约束同一提交内维度 key 唯一、同一维度作答内字段 key 唯一，并继续保留原始评级、原始分数和计算分用于审计与复算。按 ADR-0069，当前仍处开发阶段，本次切换直接重置旧绩效业务数据，不保留旧评估项结果兼容模型。
