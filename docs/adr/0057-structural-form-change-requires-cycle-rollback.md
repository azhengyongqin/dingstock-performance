# 有提交后的表单结构变化必须先退回草稿

周期内相关表单已经存在正式提交后，仍可修改维度权重、评级区间、映射分、约束规则等计算配置并统一重算，也可修正文案、说明与占位提示；但不能在 `ACTIVE` 状态直接新增或删除评估项、修改计分项类型、必填规则或维度归属。确需结构性修改时，只能由超级管理员先把周期退回 `DRAFT`，按 ADR-0053 使旧校准与结果链失效并解除评估锁定，再修改表单结构和重新激活收集。

结构变化后，已有填写内容按稳定评估项标识匹配：仍存在且数据类型兼容的内容可以保留；无法匹配或类型不兼容的旧内容不物理删除，但不进入新表单和新计算。

**Considered Options**：在进行中周期直接改变表单结构并自动迁移，可以减少回退步骤，但新增必填项、删除计分项或改变类型会使既有提交不再完整，并可能在不通知填写人的情况下改变答卷语义。要求先整体退回能明确使旧结果失效并重新开放填写。

**Consequences**：周期配置编辑器需要区分计算修改、非结构文案修改和结构修改。存在提交时，结构操作入口只在退回草稿后开放。评估项必须有跨周期配置版本稳定标识，旧值迁移需要类型兼容检查与孤立值保留机制。
