# 默认 360°维度聚焦同级员工可观察行为

默认普通岗 D 的 360°子表单采用三个维度：工作贡献与责任担当 35%（核心维度）、协作沟通与价值观 45%、学习成长与潜力 20%。默认管理岗 M 同样采用三个维度：结果推动与责任担当 40%（核心维度）、协作沟通与组织影响 35%、领导力与价值观 25%。两类表单默认使用加权评级。

360°评审员均为同级员工，因此维度只要求评价日常合作中可观察到的贡献、责任、协作、影响和价值观表现。M 类“结果推动”只评价跨团队事项推动、承诺兑现和问题闭环，不要求掌握完整业务指标；“领导力”只评价可观察到的担当、判断和以身作则，不评价团队内部人才培养细节。

**Considered Options**：完全复用上级评估的核心业绩、管理绩效和人才培养维度，便于横向比较，但同级评审员往往缺少足够信息，容易产生猜测评分。只设置一两个综合维度虽更省事，但反馈区分度过低。每类保留三个可观察维度，在填写成本与参考价值之间取得平衡。

**Consequences**：系统初始化需要创建上述 D/M 默认 360°子表单及权重，并显式标记核心维度。表单说明和评级提示必须限制评价边界，避免把同级环评解释为完整业绩或管理审计。Admin仍可通过新表单模板版本调整默认结构。
