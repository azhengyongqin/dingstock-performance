# 评级映射分可配置并随周期固化

加权评级模式需要把评审人选择的评级转换为统一计算分。每个评级因此拥有可配置的评级映射分，默认 S=95、A=85、B=70、C=50。映射分必须在 0～100 内，并落在该评级自己的分数区间中；评级区间变化时必须重新校验映射分。配置模板中的映射分在创建或重新套用绩效周期时复制为评估规则快照，后续模板修改不影响已创建周期。

**Considered Options**：把映射分固定在代码中实现最简单，但评级区间或企业口径调整时会产生无法配置的隐含常量；自动取区间中点省去配置，却未必符合业务期望。允许管理员在受控区间内配置，可以保持灵活，同时通过区间约束避免明显不一致。

**Consequences**：现有评级 JSON 需要增加 `mappingScore`，模板与周期规则编辑器必须联合校验区间连续性和映射分归属。评级型评估项提交时既保存原始评级，也保存按周期快照得到的计算分；计算明细应记录使用的评级映射参数。
