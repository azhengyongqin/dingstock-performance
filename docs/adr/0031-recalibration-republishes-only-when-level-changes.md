# 重新校准仅在等级变化时重新推送确认

结果尚未推送时，最新一次成功校准直接成为待推送结果。结果已经推送后，如果显式重新校准导致员工可见最终等级变化，系统必须生成新的结果版本：旧结果版本、推送记录和确认记录永久保留，旧的未完成确认待办失效，新结果重新推送并要求员工重新确认。员工此前已经确认，也必须对新版本重新确认。申诉改判遵循同一版本规则并进入再次确认。若新校准仍维持原等级，虽然保留新的校准审计记录，但不生成新的员工结果版本，也不重开确认。周期归档后禁止重新校准或直接修改历史结果。

**Considered Options**：任何重新校准都要求员工重新确认，会让仅补充审计记录或再次明确“维持”的操作产生无意义打扰；而等级变化后仍沿用旧确认，则会让员工的确认对象与当前结果不一致。以员工可见等级是否变化作为版本与重新确认边界，可以保证确认具备明确对象，同时控制不必要的通知。

**Consequences**：员工结果、推送和确认都必须关联不可变的结果版本，不能只在参与者记录上覆盖一个当前等级。生成新版本时需要原子地失效旧待办、创建新待办并发送通知；旧确认继续用于审计但不代表员工接受新版本。归档前置检查以当前生效结果版本是否闭环为准。归档后的纠错只能进入另行定义的受控流程，不能复用普通重新校准接口。
