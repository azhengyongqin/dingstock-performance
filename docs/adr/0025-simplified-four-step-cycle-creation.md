# 绩效周期使用四步简化创建流程

默认创建流程固定为四步：基本信息（周期名称、已发布配置模板版本、计划启动时间）、参与者（选人、职级前缀检查、晋升评估标记）、日程预览（生成并可调整阶段窗口与通知规则）、启动检查（预览匹配和完整性后保存草稿或启动）。评估规则、评级映射、约束、360°关系权重、维度和评估项不再作为默认向导的必经步骤；确需修改周期快照时从“高级配置”进入。

**Considered Options**：继续把规则、维度、表单和时间分别做成创建向导步骤，能让 HR 每次完全检查配置，却重复暴露配置模板已经发布校验过的内容，使日常建周期流程冗长。四步向导保留周期特有的信息和启动风险检查，把低频规则调整收敛到高级配置。

**Consequences**：当前周期编辑向导中的维度与评估规则步骤需要移出默认导航；模板选择后必须提供清晰的配置摘要和高级配置入口。启动检查仍需校验周期快照本身，不能因为来源模板已发布就跳过人员匹配、实际窗口和运行时完整性检查。
