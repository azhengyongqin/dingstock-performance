# 配置模板与评估表单模板采用不可变发布版本

配置模板和评估表单模板都采用草稿、已发布、已归档的版本生命周期。草稿允许不完整并可自由编辑，但不能用于创建周期或被正式配置引用；发布前必须通过完整性校验，发布后版本不可原地修改；修改已发布模板会复制出新的草稿版本。配置模板版本绑定具体的已发布评估表单模板版本，表单模板发布新版时不会自动更新已有配置模板。周期创建时选择已发布配置模板版本并复制完整快照。

**Considered Options**：继续使用“可用/不可用”而允许模板原地编辑，模型更简单，但共享表单模板变化会让同一个配置模板在不同时间创建出不同内容，难以审计。让配置模板始终跟随表单模板最新版能减少升级操作，却会产生隐式行为变化。不可变版本与显式升级增加少量管理步骤，换来可复现和可追踪的周期配置来源。

**Consequences**：数据库需要稳定模板标识、版本记录、发布状态、版本号和来源版本关系；配置模板到评估表单模板的关联必须指向具体版本。模板管理页需要草稿编辑、发布校验、从已发布版本创建新草稿、归档和版本历史能力；旧的 `canCreateCycle` 仅可作为发布前校验结果，不能再替代发布状态。
