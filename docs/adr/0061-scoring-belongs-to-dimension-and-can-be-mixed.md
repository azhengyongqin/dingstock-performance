# 计分能力属于评估维度并允许混合计分方式

评估表单采用“评估维度 + 表单字段”两层结构：计分维度直接拥有评级或 0～100 分数计分方式，非计分的文本、选项、附件和链接作为表单字段归属维度，不再存在评级或分数评估项。同一加权子表单允许不同计分维度混用评级与分数；评级先按当期规则转换为 0～100 计算分，再与原始分数一起按维度占比汇总，因此阶段级“加权评级/加权评分”合并为“加权计算”。这样避免为每个维度制造唯一计分子项，也使管理员配置结构与填写人看到的评价单元一致。

**Considered Options**：保留子表单统一计分方式改动较小，但会让计分方式实质仍属于子表单，在每个维度重复展示且限制不同评价内容选择合适控件；允许维度包含多个计分输入则需要引入第二层权重和缺项归一化。维度直接产生一个计分结果，同时允许维度间混用计分方式，保留了单层权重和统一计算链。

**Consequences**：阶段计算、约束配置、模板发布校验和结果快照必须按维度读取计分方式；计分输入不提供必填开关，草稿可以缺失，正式提交时所有计分维度必须完成。非计分维度不拥有计分方式、占比或核心标记，并允许不包含表单字段，此时仅作为说明内容区展示且不产生作答。旧的计分评估项与阶段级加权评级/加权评分需要迁移，历史原始评级、原始分数与计算分仍须保持可解释和可复算。
