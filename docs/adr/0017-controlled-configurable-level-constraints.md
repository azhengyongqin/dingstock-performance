# 等级约束采用固定类型与可配置参数

核心维度与任一维度的低分封顶规则允许管理员在配置模板中调整参数，但系统只提供受控的规则类型，不接受自定义公式或脚本。默认评级模式为“核心维度 C 强制 C、核心维度 B 最高 B、任一维度 C 最高 B”；默认评分模式为“核心维度低于 60 强制 C、核心维度低于 80 最高 B、任一维度低于 60 最高 B”。红线认定强制 C 属于系统硬约束，不允许模板关闭或修改。

**Considered Options**：完全写死约束最简单，却无法适应未来模板或评级体系调整；允许任意表达式最灵活，但会引入脚本安全、规则调试、版本兼容和结果解释风险。固定规则类型配合结构化参数，可以覆盖明确业务变化，同时保持校验、计算和审计可预测。

**Consequences**：评估规则快照需要结构化保存启用的约束类型、阈值、目标等级和适用模式；模板编辑器必须使用受控表单并即时校验，不能提供代码编辑器。计算结果需要逐条记录命中的约束、约束参数和应用前后等级，方便校准与审计解释。
