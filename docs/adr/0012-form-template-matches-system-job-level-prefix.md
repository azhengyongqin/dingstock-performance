# 评估表单模板按系统职级前缀唯一匹配

普通岗与管理岗评估表单不能依赖管理员自由输入的“岗位分组”字符串。每份评估表单模板创建时必须选择公司系统中的职级前缀，当前为 D 与 M；周期添加参与者及启动复核时，使用“组织架构 > 成员”表格展示的 CoreHR `job_level.name` 职级名称解析参与者职级快照前缀并唯一匹配表单模板，多语言名称的选取顺序与成员表格一致（中文、英文、首个值），仅旧同步记录完全没有 `name` 字段时兼容 `job_level.code`；`name` 为空数组或格式错误仍按成员表格口径视为职级缺失。同一配置模板选用的表单模板不得声明重叠前缀。业务上员工职级永不为空，因此系统不提供兜底模板或日常手工指定入口；一旦启动检查发现职级缺失、前缀未覆盖或多重匹配，应将其作为主数据或模板配置错误阻止启动并要求修正源数据。

**Considered Options**：保留自由文本 `jobCategory` 或允许 HR 为异常人员临时选择模板，短期更宽松，但会掩盖职级同步或模板覆盖错误，使同职级员工拿到不同口径的表单。依赖受控的系统职级前缀并在启动前强校验，能保证匹配确定且可审计。

**Consequences**：模板设计器需要从受控职级前缀列表选择适用范围，不能手输岗位分组；配置模板可用性、参与者添加和周期启动检查必须验证前缀不重叠和参与者覆盖完整。`jobLevelSnapshot` 保存用于成员表格展示和匹配的完整 CoreHR 职级对象；自由文本 `applicableScope.jobCategory` 不能作为未经校验的运行时事实。
