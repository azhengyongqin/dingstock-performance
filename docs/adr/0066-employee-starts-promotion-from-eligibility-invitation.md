# 员工通过资格通知自行发起独立晋升流程

晋升是独立于绩效周期的流程，不嵌入员工自评、360°评估或上级评估，也不是绩效周期归档前必须完成的节点。系统只在包含本期最终绩效等级的不可变结果版本正式发布并推送给员工时检查资格：当前结果与该员工紧邻的上一半年度绩效周期有效最终结果必须分别属于 A 或 S，A+A、A+S、S+A、S+S 均符合；不能跳过中间周期查找更早的 A/S，上一周期未参与、无绩效结果、中途退出或缺少有效最终等级都不满足。符合时在发送本期绩效结果的同时发送晋升资格通知，尚未发布的校准结果不触发判断。资格通知只提供申请入口，不自动创建晋升流程；员工选择参与并提交晋升申请表单后，才正式发起晋升流程。

**Considered Options**：由 Leader 决定是否进入晋升会让员工缺少主动申请权；在绩效周期创建时预先标记无法使用尚未形成的本期等级；把晋升设为绩效周期内节点则会让未完成晋升阻塞绩效归档。结果发布时只发资格邀请、由员工提交申请后创建独立流程，能够同时满足连续绩效门槛、员工意愿和流程解耦。

**Consequences**：绩效评估表单模板只包含员工自评、360°评估和上级评估；原晋升表单内容与历史数据只读保留，退出新的绩效模板发布校验、周期快照和三类评估提交链，后续晋升专项重构时再迁移为独立晋升申请表单模板。资格判断需要读取当前及上一连续周期的最终绩效等级，并在结果版本发布事务中产生关联该版本的可审计资格邀请，再与结果通知一起发送。每次发布新结果版本都重新校验：仍满足时沿用现有资格，不重复创建申请；首次满足时补发邀请；不再满足时撤销未使用入口，已提交申请则终止晋升流程，同时保留邀请、申请和终止原因并通知员工。未申请、放弃或后续晋升流程未完成都不阻塞绩效周期归档，也不能改写原绩效结果。
