# 晋升是绩效完成后的独立流程

> 本决定先被 ADR-0065 取代，现由 ADR-0066 再次修正：晋升是员工收到绩效结果与资格通知后自行申请发起的独立流程。

晋升不再作为绩效评估表单模板中的第四个子表单，也不嵌入员工自评或上级评估提交。晋升应在绩效流程完成后独立发起，本期最终绩效等级作为是否具备参与晋升流程资格的条件之一；晋升拥有独立的流程、表单、提交和结果，不能反向改变已经完成的绩效阶段结果。

**Considered Options**：继续把晋升材料和 Leader 结论嵌入绩效评估任务可以复用现有提交入口，但会在绩效等级尚未确定时提前收集晋升内容，也无法用本期最终等级作为准入条件。独立流程使绩效结果成为清晰前置条件，并允许后续单独设计晋升角色、节点和计算规则。

**Consequences**：绩效评估表单模板只包含员工自评、360°评估和上级评估三个子表单；绩效周期参与者不再配置晋升标记，三类绩效评估提交不再携带晋升内容。旧晋升子表单和相关数据在迁移时必须与未来晋升领域隔离，不能静默混入新的绩效计算或新晋升流程。
