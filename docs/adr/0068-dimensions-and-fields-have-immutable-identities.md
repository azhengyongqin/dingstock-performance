# 维度与字段使用跨版本不可变标识

每个评估维度和表单字段在创建时获得不可变业务 key；从已发布版本复制新草稿时继承原 key，重命名、修改说明、改变类型、调整顺序或移动字段都不改变 key，新增对象才生成新 key，删除后永不复用。类型改变可能使旧值不兼容，但这是同一对象的结构变化，由兼容性校验决定是否预填，不能通过更换 key 隐藏。数据库自增 ID 只作为单个版本内的关系主键，不承担跨版本身份。周期快照、维度作答、字段作答、结构差异和兼容预填全部使用不可变 key 匹配。

**Considered Options**：继续用子表单、填写对象和排序位置拼接 key 实现简单，但换序会改变身份，甚至把旧答案错误匹配到新内容；直接复用数据库 ID 又无法跨版本复制保持一致。独立不可变 key 将内容身份与展示位置、版本行身份解耦。

**Consequences**：模板维度与字段需要持久化唯一业务 key，并在版本复制时显式继承；同一表单模板谱系内不能复用 key。快照生成不得再从 sortOrder 派生 key。历史位置 key 迁移后，后续所有结构变更和作答迁移必须按新 key 判断，不能用名称或排序猜测身份。
