# 条件必填规则属于具体表单字段

全局评级定义不再保存“该评级要求评语”的布尔配置，因为一个维度可能没有评语字段，也可能包含多个文本字段，无法确定约束目标。表单字段通常配置为选填或始终必填；计分维度中的多行文本与 Markdown 字段还可以指定在 S/A/B/C 哪些维度等级下条件必填。评级维度直接使用所选等级，分数维度先按当期评级区间派生等级，再执行字段完整性校验。非计分维度没有等级，只能使用选填或始终必填。

**Considered Options**：继续在评级定义中维护评语必填改动较小，但规则无法稳定绑定具体字段；把维度内第一个文本字段隐式当作评语又会依赖顺序并产生意外。由字段显式声明条件等级，使管理员能看到规则作用对象，也适用于混合计分和分数维度。

**Consequences**：评级配置、API 和编辑器删除 `commentRequired`；表单字段增加受控必填规则，只有计分维度中的多行文本或 Markdown 可以配置等级条件。模板发布必须校验条件等级合法，正式提交必须在服务端按维度最终输入派生等级并校验命中的字段，不能只依赖前端提示。
