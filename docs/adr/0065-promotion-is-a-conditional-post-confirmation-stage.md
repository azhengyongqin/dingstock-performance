# 晋升是最终绩效确认后的条件节点

> 本决定已被 ADR-0066 取代：晋升不是绩效周期内节点，也不由 Leader 决定进入，而是符合资格的员工自行申请发起的独立流程。

晋升不是独立流程，也不嵌入员工自评或上级评估，而是同一绩效周期中位于员工确认最终绩效之后的条件节点。只有最近两次已确认最终绩效等级均为 A 或 S 的参与者具备准入资格，再由其直属 Leader 决定是否继续进入晋升节点；未满足资格或 Leader 决定不进入时跳过。评估表单模板继续保留晋升评估子表单，进入节点后才使用，晋升结果不回写已经确认的绩效等级。

**Considered Options**：把晋升嵌入自评和上级评估会在本期最终等级尚未确定时提前收集材料；拆成完全独立流程又割裂了同一周期内从绩效结果到晋升判断的连续进度。确认绩效后再按历史等级准入，并由直属 Leader 作最终进入决定，可以保持流程连续，同时确保晋升只面向满足硬条件的员工。

**Consequences**：周期创建时不再预先勾选参与者是否晋升；系统需要在最终绩效确认后读取最近两次已确认最终等级，向符合资格者的直属 Leader 提供进入或跳过晋升节点的决策。晋升表单、提交、结果和进度必须与前三类人工评估分开建模，且不能改变已确认绩效结果。
