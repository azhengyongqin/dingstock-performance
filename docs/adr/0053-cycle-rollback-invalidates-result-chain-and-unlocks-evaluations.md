# 周期退回使结果链失效并解除评估锁定

超级管理员把 `ACTIVE` 周期退回 `SCHEDULED` 或 `DRAFT` 时，所有现有校准结果、员工结果版本、确认记录和进行中的申诉都标记为因周期退回而失效，但永久保留历史且不物理删除。所有已校准参与者解除评估锁定并回到 `ACTIVE`；退回期间任务只读，周期重新激活后，员工、360°评审员和考核 Leader可以继续修改或重新提交。已有答卷内容、评审员指派和临时草稿全部保留。重新激活后，所有需要绩效结果的参与者必须重新满足员工自评与上级评估前置条件并重新校准。中途退出记录默认保持；本周期无绩效结果仍可按原规则撤销。

已推送结果的员工必须收到通知，明确旧结果、旧确认和旧申诉均已失效，周期将重新评估并在后续推送新结果。

**Considered Options**：只退回周期状态但保留校准和评估锁定，会使周期无法实质重做；直接删除旧结果链则失去敏感操作的审计证据。将旧链标记失效、保留输入并解锁评估，既支持整体纠错，也维持历史可追溯。

**Consequences**：校准、结果版本、确认和申诉都需要有效性或失效原因字段；查询当前结果必须排除失效链。退回需要原子或可恢复地失效结果链、重置参与者主状态、解除锁定、暂停任务并创建通知任务。重新激活不得重复创建参与者、答卷或评审员指派。
