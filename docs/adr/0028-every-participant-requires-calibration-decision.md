# 每位有绩效结果的参与者推送前必须形成校准结果

每个既未中途退出、也未按 ADR-0044 标记为本周期无绩效结果的参与者，在结果推送前都必须形成一条明确校准结果，不能只在等级发生调整时记录。校准动作分为“维持”和“调整”：维持时校准后等级等于校准前等级；调整时校准后等级不同且调整原因必填。每条记录保存操作人、时间以及所参考的阶段结果版本。后续重新校准只能追加新记录，不能覆盖历史。按 ADR-0029，直属 Leader、授权范围内 HR 和 Admin 均可形成正式校准结果。

**Considered Options**：只记录发生变化的校准操作可以减少数据量，但未调整人员的最终等级仍动态依赖上级评估阶段结果，无法建立 ADR-0027 所需的人工结果边界。为所有参与者落一条明确决定，能区分“尚未校准”和“校准后决定维持”，也让结果推送拥有稳定来源。

**Consequences**：现有只描述等级调整的 `PerfCalibration` 需要扩展为完整校准决定，增加维持/调整动作及参考阶段结果版本；校准记录继续 append-only。结果推送前置检查必须确保所有需要绩效结果的参与者均存在有效的最新校准结果，批量维持原等级需要作为高效操作提供。
