# 绩效周期只保留四个粗粒度生命周期状态

绩效周期状态固定为 `DRAFT`（草稿）、`SCHEDULED`（待启动）、`ACTIVE`（进行中）和 `ARCHIVED`（已归档）。删除周期级 `SELF_REVIEW`、`REVIEWING`、`AI_ANALYZING`、`CALIBRATING`、`CONFIRMING` 等互斥阶段状态。同一进行中周期允许不同参与者分别处于评估、校准、结果确认或申诉进度；页面展示的“当前阶段”根据参与者进度聚合，仅用于运营观察，不作为权限和状态机依据。各任务按配置开始时间开放，填写提醒截止时间仅触发通知，开放后直到参与者评估锁定前可填写。AI 继续作为独立异步参考能力，不进入周期状态。`ACTIVE` 可以由超级管理员退回 `SCHEDULED` 或 `DRAFT`，即使已有校准、结果推送、确认或申诉也允许；退回必须使已推送结果失效并通知员工。

**Considered Options**：保留全局阶段状态适合所有人严格同步推进的流程，但当前规则允许逐员工校准、逐员工锁定和确认，并允许截止后继续填写，单一周期不可能准确处于某一个细粒度阶段。粗粒度生命周期加参与者级进度能如实表达并行状态，避免为了推进少数员工而错误关闭其他人的任务。

**Consequences**：现有周期状态枚举和迁移逻辑需要重构；周期级操作权限只依赖是否草稿、待启动、进行中或已归档。仪表盘、通知和归档校验改为聚合参与者、任务、提交、校准、结果版本和申诉状态。AI 相关周期状态与推进接口应删除。
