# 正式评估提交必须完整且不可变（已被 ADR-0037 部分取代）

> 状态：关于“正式提交不可变、退回后创建修订版本”的决定已被 ADR-0037 取代；正式提交完整性校验继续有效。

评估草稿允许不完整。正式提交员工自评、360°评估或上级评估时，所有常规维度的唯一计分项都必须填写，模板标记为必填的非计分项也必须填写；员工自评等级始终必填。参与者启用晋升评估时，员工侧晋升必填项随员工自评一起校验，Leader 侧晋升必填项随上级评估一起校验，360°评审员不展示也不填写晋升内容。填写提醒截止时间不豁免完整性校验；上级评估必须实际完成，不能以缺项替代。提交后的可修改性与存储方式改按 ADR-0037 执行。

**Considered Options**：允许缺少部分维度仍提交需要在维度层继续做权重归一化，并会让不同评审员按不同计算口径进入同一关系平均。直接覆盖已提交答卷则无法还原阶段结果为何变化。将不完整状态限定为草稿、正式提交要求完整并版本化，能够保持计算口径一致和审计链完整。

**Consequences**：提交接口必须基于该参与者实际生效的周期表单快照执行服务端完整性校验，不能只依赖前端必填提示。评估提交与评估项结果需要修订版本、当前版本标识和来源版本关系；阶段结果必须引用具体提交版本。退回后产生的新版本会触发相应阶段结果重算，旧版本永久只读保留。
