# 表单模型切换时重置开发环境绩效数据

项目仍处开发阶段，本次从“评估项结果”切换到“维度作答 + 字段作答”时不实施旧绩效数据迁移或双模型兼容，而是删除全部 `perf_*` 绩效业务记录以及关联的操作日志和导出任务，再通过更新后的基线脚本重建默认 D/M 表单模板、默认配置模板和开发基线周期。飞书同步 `lark_*` 数据、`role_grants` 权限授权和 `system_configs` 系统配置不在清理范围内。

**Considered Options**：非破坏性回填能够保留现有答卷和结果，但需要长期兼容错误的评估项层级、位置 key 和阶段模式，显著增加本次重构复杂度；当前数据仅用于开发验证，没有生产审计价值。限定清理绩效业务域并保留主数据、权限和系统配置，可以用更小风险建立干净的新基线。

**Consequences**：执行前必须精确限定清理表集合，不能清空整个 `performance` schema；清理后旧模板、周期、答卷、结果、校准、申诉和相关日志不可恢复，除非另有数据库备份。Schema、发布校验、快照、默认数据和 `seed-baseline-data.ts` 必须在同一次切换中完成，验证基线脚本可重新建立可用开发数据。
