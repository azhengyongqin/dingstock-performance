# 校准拒绝基于旧人工评估输入提交

校准页面需要记录打开时看到的员工自评、全部有效 360°提交、上级评估及其阶段结果的当前修订标识，以及有效红线状态。校准人员提交决定时，服务端在同一事务内重新检查：任何人工评估在此期间重新提交，或有效红线状态发生变化，都拒绝本次校准并要求刷新查看最新差异。首次校准校验成功后，校准结果写入与该参与者评估锁定必须原子完成。AI 报告是可缺失、可重跑的异步参考信息，其更新不使校准提交失效。

**Considered Options**：只检查上级评估阶段结果可以保护校准前权威等级，但自评和 360°同样是校准人员会参考的信息，使用旧参考内容提交仍可能造成错误判断。把 AI 报告也纳入并发令牌则会让后台异步任务无谓阻断人工操作。检查全部人工评估与红线、排除非权威异步 AI，能覆盖实质输入变化且保持流程可用。

**Consequences**：参与者需要一个随人工评估正式重新提交或红线状态变化而递增的校准输入修订号，或提供等价的组合版本校验。校准请求携带期望修订号和期望最新校准标识；服务端以原子条件检查、写入校准并锁定评估。冲突响应需要返回发生变化的数据类别，前端要求刷新，不得自动重试。
