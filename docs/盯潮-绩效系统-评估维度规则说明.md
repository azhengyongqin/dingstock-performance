# 盯潮-绩效系统-评估维度规则说明

## 评估规则

### 统一维度加权计算

员工自评、360°评估和上级评估固定使用维度加权计算，不再配置阶段级“直接定级、加权评级、加权评分”模式。同一子表单可以同时包含评级计分维度和 0～100 分计分维度：

- 评级维度按当期评级配置的 `mappingScore` 转为 0～100 计算分。
- 分数维度的原始分数直接作为计算分。
- 每个计分维度再按当期连续评级区间派生维度等级。
- 计分维度占比必须大于 0%、不超过 100%、最多两位小数；同一子表单合计严格为 100%，并且有且只有一个核心维度。

绩效等级区间、映射分、核心维度约束和红线规则见[绩效等级定义和计算方式](./盯潮-绩效系统-绩效等级定义和计算方式.md)。

### 评估阶段

- 员工自评：默认一个占比 100% 的评级核心维度，并可配置总结、规划等表单字段。
- 360°评估：按评审员关系聚合后再应用维度占比；默认维度使用评级计分。
- 上级评估：默认维度使用 0～100 分计分。
- AI 报告：根据人工评估及其它受控信息给出建议等级，不属于人工评估阶段，也不配置人工计分模式。

## 评估模板

评估模板统一使用“评估维度 + 表单字段”两层结构：

- **计分维度**直接定义评级或分数计分方式、占比及核心标记；正式提交时计分输入固定必填。
- **非计分维度**只用于组织说明和表单字段，不拥有计分方式、占比或核心标记，也允许只展示说明而不包含字段。
- **表单字段**只承载短文本、长文本、Markdown、单选、多选、附件和链接等非计分内容；评级与分数不是表单字段。
- 维度和字段使用跨版本稳定业务 key；改名、换序、改类型或字段跨维度移动不改变身份，删除后的 key 不得复用。

公司默认按员工职级提供两套模板：

- 普通岗评估模板（职级前缀 `D`）
- 管理岗评估模板（职级前缀 `M`）

绩效模板只包含 SELF、PEER、MANAGER 三个人工子表单。旧晋升定义及历史作为只读待迁移内容单独展示，不进入模板发布、周期快照或绩效提交链。
