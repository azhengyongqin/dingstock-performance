# AI-Service-Center 初始化部署

本文只用于服务器第一次部署。后续发布请看 [更新发布.md](./更新发布.md)。

## 部署结果

- 项目目录：`/root/dingstock/dingstock-performance`
- 访问地址：`http://8.137.151.95/performance`
- 后端地址：`http://8.137.151.95/performance/backend/`
- Swagger：`http://8.137.151.95/performance/backend/api-docs`
- PM2 配置：[pm2/ecosystem.config.cjs](./pm2/ecosystem.config.cjs)
- Caddy 配置：[caddy/Caddyfile](./caddy/Caddyfile)

## 1. 前置检查

服务器需提前安装：

- Node.js `22.23.1`
- pnpm
- PM2
- Caddy

本地确认 SSH 可用：

```bash
ssh AI-Service-Center \
  'export PATH=/root/.local/share/pnpm/bin:/root/.nvm/versions/node/v22.23.1/bin:$PATH; node -v; pnpm -v; pm2 -v; caddy version'
```

成功标志：四个命令都能输出版本号。

## 2. 上传第一个 release

在本地项目根目录执行：

```bash
export PERFORMANCE_RELEASE_ID=$(date +%Y%m%d%H%M%S)
echo "$PERFORMANCE_RELEASE_ID"

ssh AI-Service-Center \
  "mkdir -p /root/dingstock/dingstock-performance/releases/$PERFORMANCE_RELEASE_ID"

git archive --format=tar HEAD | ssh AI-Service-Center \
  "tar -xf - -C /root/dingstock/dingstock-performance/releases/$PERFORMANCE_RELEASE_ID"
```

记住终端输出的 `PERFORMANCE_RELEASE_ID`。

成功标志：服务器 release 目录中包含 `backend`、`web`、`deploy`。

## 3. 创建生产 YAML

登录服务器，并把 `<release-id>` 替换为上一步的编号：

```bash
ssh AI-Service-Center

export PERFORMANCE_ROOT=/root/dingstock/dingstock-performance
export PERFORMANCE_RELEASE_ID=<release-id>

mkdir -p "$PERFORMANCE_ROOT/shared"
cp "$PERFORMANCE_ROOT/releases/$PERFORMANCE_RELEASE_ID/deploy/config/backend.prod.example.yaml" \
  "$PERFORMANCE_ROOT/shared/backend.prod.yaml"
chmod 600 "$PERFORMANCE_ROOT/shared/backend.prod.yaml"

vi "$PERFORMANCE_ROOT/shared/backend.prod.yaml"
```

填写 PostgreSQL、Redis 和飞书配置。真实密码只能放在服务器 YAML，不要提交到 Git。

如果生产环境需要开启快速员工登录，还要同时填写：

```yaml
auth:
  devLogin:
    enabled: true
    password: '<恰好32位的强随机密码>'
```

成功标志：`enabled` 为 `true`，并且替换后的 `password` 恰好是 32 位；否则后端会拒绝启动。

让 release 使用这份共享配置：

```bash
mv "$PERFORMANCE_ROOT/releases/$PERFORMANCE_RELEASE_ID/backend/config/prod.yaml" \
  "$PERFORMANCE_ROOT/releases/$PERFORMANCE_RELEASE_ID/backend/config/prod.yaml.repo-default"

ln -s ../../../../shared/backend.prod.yaml \
  "$PERFORMANCE_ROOT/releases/$PERFORMANCE_RELEASE_ID/backend/config/prod.yaml"
```

成功标志：

```bash
readlink -f "$PERFORMANCE_ROOT/releases/$PERFORMANCE_RELEASE_ID/backend/config/prod.yaml"
```

输出：`/root/dingstock/dingstock-performance/shared/backend.prod.yaml`。

## 4. 安装依赖并构建

继续在服务器执行：

```bash
export PATH=/root/.local/share/pnpm/bin:/root/.nvm/versions/node/v22.23.1/bin:$PATH

cd "$PERFORMANCE_ROOT/releases/$PERFORMANCE_RELEASE_ID/backend"
CI=true pnpm install --frozen-lockfile
pnpm build

cd "$PERFORMANCE_ROOT/releases/$PERFORMANCE_RELEASE_ID/web"
CI=true pnpm install --frozen-lockfile
CI=true \
BASEPATH=/performance \
NEXT_PUBLIC_API_BASE_URL=http://8.137.151.95/performance/backend \
NEXT_PUBLIC_APP_URL=http://8.137.151.95/performance \
pnpm build
```

成功标志：后端生成 `dist/src/main.js`，前端显示 `Compiled successfully`。

## 5. 初始化数据库并启动 PM2

```bash
cd "$PERFORMANCE_ROOT/releases/$PERFORMANCE_RELEASE_ID/backend"
NODE_ENV=production pnpm exec prisma migrate deploy

ln -sfn "releases/$PERFORMANCE_RELEASE_ID" "$PERFORMANCE_ROOT/current"

cd "$PERFORMANCE_ROOT/current"
pm2 start deploy/pm2/ecosystem.config.cjs
pm2 save
```

成功标志：

```bash
pm2 status
```

`dingstock-performance-backend` 和 `dingstock-performance-web` 均为 `online`。

如果需要初始化默认表单、配置模板和绩效周期，再执行一次：

```bash
cd "$PERFORMANCE_ROOT/current/backend"
NODE_ENV=production pnpm seed:baseline
```

成功标志：输出 `基线数据初始化完成`。

## 6. 启用 Caddy 路由

```bash
mkdir -p /etc/caddy/routes

install -m 644 "$PERFORMANCE_ROOT/current/deploy/caddy/Caddyfile" \
  /etc/caddy/Caddyfile

install -m 644 "$PERFORMANCE_ROOT/current/deploy/caddy/routes/performance.caddy" \
  /etc/caddy/routes/performance.caddy

caddy fmt --overwrite /etc/caddy/Caddyfile
caddy fmt --overwrite /etc/caddy/routes/performance.caddy
caddy validate --config /etc/caddy/Caddyfile
systemctl enable --now caddy
```

成功标志：看到 `Valid configuration`，且 `systemctl is-active caddy` 输出 `active`。

## 7. 最终验收

退出服务器，在本地执行：

```bash
curl -L -o /dev/null -w '%{http_code}\n' http://8.137.151.95/performance
curl -o /dev/null -w '%{http_code}\n' http://8.137.151.95/performance/backend/
curl -o /dev/null -w '%{http_code}\n' http://8.137.151.95/performance/backend/api-docs
```

成功标志：三项均输出 `200`。
