# 测试员工自评

用 Cursor Browser 打开 `http://localhost:3001/self-review`，按以下顺序填写所有字段并保存草稿。

## 操作流程

```
- [ ] 1. browser_navigate 打开页面
- [ ] 2. browser_take_screenshot 确认页面已加载
- [ ] 3. browser_lock 锁定标签
- [ ] 4. 选择自评等级
- [ ] 5. 填写自评总结
- [ ] 6. 填写半年度总结
- [ ] 7. 填写下个半年规划
- [ ] 8. 填写需要的支持和帮助
- [ ] 9. 点击「保存草稿」
- [ ] 10. 等待 2 秒后截图确认保存成功
- [ ] 11. browser_lock unlock
```

## 字段填写规范

### 自评等级
选择 `A · 优秀`（radio 选项）。

### 自评总结（e29）
填写约 200～300 字，结合后端开发实际工作内容，示例：

```
本半年度，我在产品研发、跨团队协作和个人能力提升三个维度均超出预期目标。在核心业务模块的开发中，独立完成了绩效系统后端 API 的设计与实现，涵盖组织架构同步、绩效周期管理、评估表单引擎等关键能力，整体代码质量高，测试覆盖率达 85%+，未出现线上故障。同时积极参与跨部门需求对齐，推动多个模块按时交付，获得产品和设计同学的正向反馈。在个人成长方面，深入掌握了 NestJS + Prisma + PostgreSQL 的生产级实践，自发建立了团队代码规范文档，得到团队认可并沉淀为标准。综合来看，本半年工作表现整体超出预期，自评 A·优秀。
```

Markdown 格式，包含工作产出和个人成长两个大节：

## 保存确认

点击「保存草稿」后：
- 按钮短暂变为 `disabled` = 请求发送中
- 按钮恢复可用 = 保存成功
- 截图后 unlock 浏览器，向用户汇报填写结果
