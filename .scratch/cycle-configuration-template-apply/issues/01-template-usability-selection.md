# 配置模板可用性贯穿模板选择

Status: ready-for-agent

## Parent

.scratch/cycle-configuration-template-apply/PRD.md

## What to build

让配置模板列表和绩效周期创建入口都能区分“模板存在”和“可用于创建”。后端为每个未删除配置模板返回是否可用于创建及不可用原因；前端在周期创建基础信息步骤只默认选中“默认且可用”的配置模板，并把不可用模板置灰展示原因。模板管理仍允许创建和编辑不完整模板。

## Acceptance criteria

- [x] 模板列表能表达每个配置模板是否可用于创建绩效周期，并给出业务可读的不可用原因。
- [x] 缺少评分等级、缺少评估维度、维度权重按适用分组不完整的配置模板都不可用于创建。
- [x] 新建绩效周期时，只自动选中“默认且可用”的配置模板；默认但不可用时不自动选择。
- [x] 新建绩效周期模板下拉中，不可用模板可见但不可选，并展示不可用原因。
- [x] 模板管理仍允许保存不完整配置模板，方便 HR 分阶段维护。
- [x] 覆盖配置模板可用性计算的后端测试，并覆盖默认可用模板选择/不可用模板禁用的前端可观察行为。

## Blocked by

None - can start immediately
